## 🚀 4. Founding Story (창업 배경)

### **"밤 11시, 어두운 골목길에서 시작된 질문"**

창업자는 퇴근길 어두운 골목에서 고장난 가로등을 발견했습니다. 신고하고 싶었지만:
- 민원24 접속 → 로그인 → 카테고리 찾기 → 주소 입력 → 상황 설명 작성... 
- **최소 5분 이상 소요되는 복잡한 절차**에 결국 포기했습니다.

다음날 뉴스에서 같은 지역에서 발생한 안전사고를 보고 깨달았습니다:
> **"신고하기 어려운 시스템이 시민 안전을 위협하고 있다"**

동시에 지자체 공무원들을 만나며 알게 된 현실:
- 하루 평균 **200~300건의 민원**을 수작업으로 분류
- 긴급 신고가 일반 민원에 묻혀 **대응 지연**
- 부정확한 위치·설명으로 인한 **재확인 업무 과부하**

**"사진 한 장이면 되지 않을까?"**

이 질문에서 시작된 우리동네 리포터는, AI가 시민과 행정을 연결하는 가장 쉬운 방법을 만들고 있습니다.

## 🛠️ 6. Product / Service Overview

### ■ 서비스 요약

우리동네 리포터는 **사진 기반 문제 인식 → 긴급성 판단 → 자동 설명문 생성 → 담당부서 라우팅 → 신고 접수**를
모두 AI가 처리하는 **민원 자동화 시스템**입니다.

### ■ 핵심 기능

#### **1. 30초 신고 프로세스**
- 사진 촬영 → 자동 분석 → 확인 → 제출 (평균 27초)
- 위치 정보 자동 추출 (GPS EXIF 데이터)
- 카테고리 자동 분류 (16개 민원 유형)

#### **2. AI Vision 기반 문제 인식**
- YOLOv8 기반 실시간 객체 감지
- 포트홀, 불법 쓰레기, 가로등 고장, 낙석 등 **23종 문제 인식**
- 정확도 **mAP@0.5: 87.3%** (자체 데이터셋 5만 장 학습)

#### **3. 긴급도 자동 판단**
- 0~100점 긴급성 스코어 산출
- 교통사고 위험(90점 이상) → 즉시 알림
- 미관 문제(30점 이하) → 일반 접수

#### **4. LLM 기반 설명문 자동 생성**
- GPT-4V 기반 상황 설명문 생성
- 예시: "가로등이 고장나 야간 보행 안전에 문제가 있습니다. 빠른 수리가 필요합니다."
- G→V→R (Generate→Verify→Regenerate) 검증 구조

#### **5. 담당 부서 자동 라우팅**
- 16개 행정 부서 자동 분류
- 도로과 / 환경과 / 건설과 / 안전관리과 등
- 정확도 **94.2%**

#### **6. 행정 시스템 연동**
- 이메일 자동 발송
- 민원24 API 연동 (개발 중)
- 담당자 모바일 푸시 알림

### ■ 기술 스택

#### **Frontend**

- 카카오맵 API

## 🧱 7. 기술 구조 / 아키텍처

### ■ Multi-Agent Architecture (LangGraph 기반)

```
[사용자] → [이미지 업로드] → [LangGraph Multi-Agent System]
                                        ↓
                              ┌─────────────────┐
                              │ Preprocess Agent│
                              │ - EXIF 추출     │
                              │ - 위치 정보     │
                              │ - 이미지 정규화 │
                              └────────┬────────┘
                                       ↓
                              ┌─────────────────┐
                              │  Vision Agent   │
                              │ - 객체 감지     │
                              │ - 문제 분류     │
                              │ - 위험 요소 식별│
                              └────────┬────────┘
                                       ↓
                    ┌──────────────────┴──────────────────┐
                    ↓                                      ↓
          ┌──────────────────┐              ┌──────────────────┐
          │Urgency Score Agent│              │ Routing Agent    │
          │ - 긴급도 점수화   │              │ - 부서 분류      │
          │ - 0~100점 산출   │              │ - 담당자 매핑    │
          └────────┬─────────┘              └────────┬─────────┘
                   └──────────────┬──────────────────┘
                                  ↓
                       ┌─────────────────────┐
                       │  Generate Agent     │
                       │  - LLM 설명문 생성  │
                       └──────────┬──────────┘
                                  ↓
                       ┌─────────────────────┐
                       │  Verify Agent       │
                       │  - 사실성 검증      │
                       │  - 정확도 체크      │
                       └──────────┬──────────┘
                                  ↓
                              [검증 통과?]
                              ↙        ↘
                           YES          NO
                            ↓            ↓
                  ┌──────────────┐  ┌────────────────┐
                  │Dispatch Agent│  │Regenerate Agent│
                  │- 신고 제출   │  │- 재생성 (최대  │
                  │- 이메일 발송 │  │  3회)          │
                  └──────────────┘  └────────┬───────┘
                                              ↓
                                        [다시 Verify]
```

### ■ 각 Agent 상세 설명

#### **1. Preprocess Agent**
- **입력**: 원본 이미지 (JPEG/PNG, 최대 10MB)
- **처리**:
  - EXIF 메타데이터 추출 (GPS, 촬영시간)
  - 이미지 리사이징 (640x640 for YOLO)
  - 좌표 변환 (WGS84 → EPSG:5179)
- **출력**: 정규화된 이미지 + 메타데이터 JSON

#### **2. Vision Agent**
- **모델**: YOLOv8-large
- **학습 데이터**: 자체 수집 5만 장 + 공공 데이터셋 3만 장
- **감지 클래스**: 23종
  - 도로: 포트홀, 균열, 싱크홀
  - 안전: 가로등 고장, 신호등 고장, 낙석
  - 환경: 쓰레기, 불법 투기, 파손된 시설물
- **처리 시간**: 평균 **0.8초** (GPU 기준)

#### **3. Urgency Scoring Agent**
- **알고리즘**: Gradient Boosting (XGBoost)
- **입력 특성** (12개):
  - 문제 유형, 위치 특성, 시간대
  - 주변 시설물(학교/병원 100m 내), 교통량
- **긴급도 구간**:
  - 90~100: 즉시 대응 (교통사고 위험)
  - 70~89: 24시간 내
  - 50~69: 3일 내
  - 0~49: 일주일 내

#### **4. Routing Agent**
- **방식**: Rule-based + ML Classification
- **정확도**: 94.2% (1,200건 테스트)
- **16개 부서 자동 분류**
- **예시**:
  - 포트홀 → 도로과
  - 가로등 → 안전관리과
  - 쓰레기 → 환경과

#### **5. Generate → Verify → Regenerate (GVR) Agent**

**Generate Agent**:
- **모델**: GPT-4V
- **프롬프트 구조**:
```
당신은 시민 민원 담당자입니다.
이미지: [이미지 임베딩]
문제 유형: {problem_type}
긴급도: {urgency_score}점
위치: {address}

다음 형식으로 민원 설명문을 작성하세요:
1. 문제 상황 (1문장)
2. 위험성/영향 (1문장)
3. 조치 필요성 (1문장)

전문적이고 객관적인 어조로 작성하되, 200자 이내로 요약하세요.
```

**Verify Agent**:
- **검증 항목** (6개):
  1. 사실 관계 일치 (이미지 vs 텍스트)
  2. 위치 정보 정확성
  3. 긴급도와 표현 일치
  4. 문법/맞춤법
  5. 길이 (100~250자)
  6. 행정 용어 적절성
- **통과 기준**: 6개 중 5개 이상 만족

**Regenerate Agent**:
- 최대 3회 재생성
- 각 재생성 시 이전 피드백 반영
- 3회 실패 시 → 수동 검토 큐 이동

#### **6. Dispatch Agent**
- **최종 패키징**:
```json
{
  "report_id": "RPT-20251031-031731",
  "timestamp": "2025-10-31T03:17:31+09:00",
  "location": {
    "address": "전북특별자치도 부안군 번산면 격포리 258-43",
    "coordinates": [126.5234, 35.6189]
  },
  "problem_type": "가로등 고장",
  "urgency_score": 82,
  "description": "가로등이 고장나 야간 보행 안전에 문제가 있습니다. 빠른 수리가 필요합니다.",
  "department": "안전관리과",
  "images": ["img_001.jpg"],
  "reporter_id": "USER_12345"
}
```
- 이메일 발송 (담당자)
- 시스템 DB 저장
- 시민 앱 푸시 알림 ("신고가 접수되었습니다")
